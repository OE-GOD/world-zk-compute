2. How It Works

The proof demonstrates that an XGBoost model inference was executed correctly without revealing the model weights or internal computations.

A

GKR Protocol

The Goldwasser-Kalai-Rothblum (GKR) protocol verifies layered arithmetic circuits. Each layer's computation is checked via sumcheck, reducing the claim from output to input.

B

Hyrax Commitments

Pedersen-based polynomial commitments bind the prover to specific input values. The verifier checks that committed inputs match the GKR transcript without seeing the raw data.

C

DAG Topology

XGBoost trees form a directed acyclic graph (DAG), not a simple chain. The DAG verifier handles multi-claim routing, atom-level fan-out, and point templates across layers.

D

Poseidon Transcript

Fiat-Shamir challenges are derived via a Poseidon sponge over BN254. This makes the proof non-interactive: anyone can verify without the original prover.
